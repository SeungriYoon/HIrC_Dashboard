📖 시스템 개요

휴머노이드 후각 디스플레이센터 통합 관리 시스템은 연구센터의 인력, 성과, 일정, 협업 현황을 한눈에 파악할 수 있는 웹 기반 대시보드입니다.

🎯 주요 목표

  • 18명 연구진의 정보와 성과를 체계적으로 관리
  • 4개 연구그룹 간의 협력 관계 시각화
  • 연구 일정과 성과를 효율적으로 추적
  • 비개발자도 쉽게 데이터를 관리할 수 있는 인터페이스 제공

📊 지식그래프 모듈

카드 뷰

  • 18명 연구자의 정보를 카드 형태로 표시
  • 그룹별 색상으로 구분
  • 카드 클릭 시 상세 정보 모달 표시
  • 이미지로 저장 기능

네트워크 뷰

  • D3.js 기반 인터랙티브 네트워크 그래프
  • 연구자 간 협력 관계를 선으로 표시
  • 줌/팬 기능으로 자유로운 탐색
  • 성과에 따른 노드 크기 자동 조절

💼 데이터 관리 모듈

성과 추가

  • 논문, 특허, 학회발표, 수상 등록
  • 연구자별 성과 자동 집계
  • 날짜와 상세 설명 기록

연구자 정보 관리

  • 역할, 소속, 키워드 수정
  • 실시간 UI 업데이트
  • 변경 이력 추적

백업 및 복원

  • 전체 데이터 JSON 백업
  • CSV 형태로 데이터 내보내기
  • 백업 파일로부터 데이터 복원

🚀 시작하기

1. 접속 방법

  1. 웹 브라우저 열기
    Chrome, Firefox, Edge 등 최신 브라우저 사용 (IE 제외)
  2. URL 접속
    GitHub Pages URL 또는 로컬 서버 주소로 접속
  3. 로딩 대기
    데이터 로딩이 완료될 때까지 잠시 대기

2. 기본 인터페이스

좌측 사이드바

  • 연구센터 통계: 전체 현황 요약
  • 필터 & 뷰: 그룹별 필터링 버튼
  • 컨트롤: 데이터 내보내기, 초기화 등
  • 검색: 연구자명, 키워드 검색

상단 탭

  • 지식그래프: 연구자 정보 시각화
  • 연구 협업: 그룹별 협력 현황
  • 일정관리: 연구 일정 캘린더
  • 성과관리: 연구 성과 현황
  • 데이터 관리: 관리자용 편집 기능
💡 팁: 처음 사용할 때는 지식그래프 탭에서 시작하여 전체 연구진을 파악한 후, 다른 탭들을 둘러보는 것을 추천합니다.

📊 지식그래프 사용법

카드 뷰 사용법

  1. 카드 뷰 선택
    '카드 뷰' 버튼을 클릭하여 카드 형태로 전환
  2. 연구자 정보 확인
    각 카드에서 이름, 역할, 소속, 키워드 확인
  3. 상세 정보 보기
    카드를 클릭하면 상세 정보 모달이 열림
  4. 그룹별 필터링
    좌측 사이드바의 그룹 버튼으로 특정 그룹만 표시

네트워크 뷰 사용법

  1. 네트워크 뷰 선택
    '네트워크 뷰' 버튼을 클릭하여 그래프 형태로 전환
  2. 그래프 탐색
    마우스 드래그로 화면 이동, 휠로 줌 조절
  3. 노드 상호작용
    연구자 노드를 드래그하여 위치 조절
  4. 협력 관계 파악
    노드 간 연결선으로 협력 관계와 강도 확인
  5. 줌 컨트롤
    우측 상단의 +, -, 홈 버튼으로 줌 조절

검색 및 필터링

검색 기능

  • 연구자 이름으로 검색 (예: "김종만")
  • 키워드로 검색 (예: "M13파지", "딥러닝")
  • 부분 일치도 지원 (예: "김"으로 김씨 성을 가진 연구자 검색)

그룹 필터링

  • 1그룹: 후각 소재 (7명)
  • 2그룹: 후각 소자 (5명)
  • 3그룹: 후각 시스템 (3명)
  • 4그룹: 후각 데이터 처리 (3명)
⚠️ 주의: 네트워크 뷰는 처음 로딩 시 시간이 소요될 수 있습니다. 연결이 느린 환경에서는 카드 뷰를 먼저 사용해 보세요.

💼 데이터 관리

🔒 관리자 전용: 이 기능은 데이터를 수정할 권한이 있는 관리자만 사용해야 합니다.

성과 추가하기

  1. 데이터 관리 탭 이동
    상단 탭에서 "데이터 관리"를 클릭
  2. 성과 유형 선택
    논문 발표, 특허 등록, 학회 발표, 수상 중 선택
  3. 기본 정보 입력
    제목, 연구자, 날짜 입력 (필수)
  4. 상세 설명 작성
    추가 설명이나 메모 입력 (선택)
  5. 성과 추가 버튼 클릭
    입력 완료 후 "성과 추가" 버튼 클릭

연구자 정보 수정하기

  1. 연구자 선택
    드롭다운에서 수정할 연구자 선택
  2. 정보 확인
    현재 정보가 폼에 자동으로 채워짐
  3. 정보 수정
    역할, 소속, 키워드 등 필요한 부분 수정
  4. 키워드 형식
    키워드는 쉼표(,)로 구분하여 입력
  5. 수정 완료
    "연구자 정보 수정" 버튼으로 저장

데이터 백업 및 복원

백업하기

  1. 전체 데이터 백업
    "전체 데이터 백업" 버튼 클릭
  2. 파일 다운로드
    JSON 파일이 자동으로 다운로드됨
  3. 파일 보관
    안전한 위치에 백업 파일 보관

CSV로 내보내기

  1. CSV 내보내기 버튼
    "CSV로 내보내기" 버튼 클릭
  2. 파일 다운로드
    연구자 정보가 CSV 파일로 다운로드
  3. 엑셀에서 편집
    다운로드된 CSV 파일을 엑셀에서 열어 편집 가능

데이터 복원하기

  1. 데이터 가져오기 버튼
    "데이터 가져오기" 버튼 클릭
  2. 백업 파일 선택
    이전에 백업한 JSON 파일 선택
  3. 자동 새로고침
    데이터 복원 후 자동으로 페이지 새로고침
⚠️ 주의사항:
  • 데이터 수정 전에 반드시 백업을 수행하세요
  • "모든 데이터 초기화"는 되돌릴 수 없으니 신중히 사용하세요
  • 중요한 변경 사항은 정기적으로 백업하세요

⚙️ 설정 변경

시스템의 시각화 설정은 data/settings.json 파일을 편집하여 변경할 수 있습니다.

노드 크기 설정

{ "visualization": { "nodeSize": { "minSize": 20, // 최소 노드 크기 "maxSize": 80, // 최대 노드 크기 "paperWeight": 5, // 논문 1편당 추가 크기 "patentWeight": 8, // 특허 1건당 추가 크기 "awardWeight": 10 // 수상 1건당 추가 크기 } } }

색상 설정

{ "visualization": { "colors": { "group1": "#E8F4FD", // 1그룹 배경색 "group2": "#FFF2E8", // 2그룹 배경색 "group3": "#F0F8E8", // 3그룹 배경색 "group4": "#F8E8F0", // 4그룹 배경색 "defaultNode": "#4ecdc4", // 기본 노드 색상 "selectedNode": "#ff6b6b", // 선택된 노드 색상 "edge": "rgba(78, 205, 196, 0.6)" // 연결선 색상 } } }

성과 목표 설정

{ "achievements": { "targets": { "papers": 30, // 논문 연간 목표 "patents": 15, // 특허 연간 목표 "conferences": 25, // 학회 발표 연간 목표 "awards": 10, // 수상 연간 목표 "cooperation": 20, // 협력 연간 목표 "education": 30 // 인력양성 연간 목표 } } }

네트워크 레이아웃 설정

{ "visualization": { "layout": { "forceStrength": -300, // 노드 간 반발력 "linkDistance": 100, // 연결선 기본 길이 "centerForce": 0.1 // 중심으로 끌어당기는 힘 } } }
💡 설정 변경 후: 설정 파일을 수정한 후 브라우저에서 새로고침(F5)하면 변경사항이 적용됩니다.

🔧 문제 해결

자주 발생하는 문제

Q: 페이지가 로딩되지 않아요

A: 다음 사항을 확인해 보세요:

  • 인터넷 연결 상태 확인
  • 브라우저 새로고침 (Ctrl+F5 또는 Cmd+Shift+R)
  • 브라우저 캐시 및 쿠키 삭제
  • 다른 브라우저에서 시도

Q: 데이터가 표시되지 않아요

A: 다음 사항을 확인해 보세요:

  • JSON 파일들이 올바른 위치(data/ 폴더)에 있는지 확인
  • JSON 파일 문법 오류 확인 (온라인 JSON 검증 도구 사용)
  • 브라우저 개발자 도구(F12)에서 오류 메시지 확인
  • 로컬 서버로 실행하는 경우 서버가 정상 작동하는지 확인

Q: 네트워크 그래프가 느려요

A: 다음 사항을 시도해 보세요:

  • 그룹 필터를 사용하여 표시되는 노드 수를 줄이기
  • 브라우저 하드웨어 가속 활성화
  • 다른 탭이나 프로그램 종료로 메모리 확보
  • 카드 뷰 사용 고려

Q: 데이터 수정이 저장되지 않아요

A: 다음 사항을 확인해 보세요:

  • 브라우저의 로컬스토리지가 활성화되어 있는지 확인
  • 시크릿/프라이빗 모드가 아닌지 확인
  • 브라우저 저장공간이 충분한지 확인
  • 정기적으로 백업 수행

브라우저 호환성

✅ 지원 브라우저

Chrome, Firefox, Edge, Safari (최신 버전)

❌ 미지원 브라우저

Internet Explorer

성능 최적화 팁

  • 정기적인 데이터 정리: 오래된 또는 불필요한 데이터 제거
  • 이미지 최적화: 연구자 프로필 이미지 크기 최적화
  • 브라우저 업데이트: 최신 브라우저 버전 사용
  • 캐시 활용: 자주 변경되지 않는 설정은 캐시 활용
🆘 추가 도움이 필요한 경우:
  • 브라우저 개발자 도구(F12)의 콘솔 탭에서 오류 메시지 확인
  • 문제 상황의 스크린샷과 함께 관리자에게 문의
  • GitHub Issues에 문제 보고 (해당하는 경우)
